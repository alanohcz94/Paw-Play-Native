FIX 1 — SCORING ENGINE: Skip rules by difficulty
Affects: calculateScore() utility — all game modes
═══════════════════════════════════════════════════════════════

Current wrong behaviour:
  Skip deducts the same penalty regardless of difficulty level.

Fix the calculateScore() function so skip logic is:

  EASY:
    skip (no exceed)       = 0 pts
    skip after exceed      = 0 pts (floor applies, no negatives)
    score floor            = 0 (display and calculation)

  MEDIUM:
    skip (no exceed)       = 0 pts
    skip after exceed      = 0 pts (floor applies, no negatives)
    score floor            = 0 (display and calculation)

  EXPERT:
    skip (no exceed)       = -20 pts
    skip after exceed      = -20 pts AND -1pt per second
                             already accumulated
    score floor            = NONE (negatives allowed and shown)

Complete scoring rules for reference — implement all of these
exactly in calculateScore():

  Complete within window   = +20 pts
  Exceed window (no skip)  = -1 pt per second over (all levels)
  Skip Easy/Medium         = 0 pts
  Skip Expert              = -20 pts
  Skip Expert + exceed     = -20 pts + -1pt/sec accumulated

Store the raw calculated value in the database always.
Apply max(0, score) only at the display/render layer for
Easy and Medium.

FIX 7 — TRAINING MODE: Variable reward schedule 49/51 random
Affects: Training mode — variable schedule session loop
═══════════════════════════════════════════════════════════════

Current wrong behaviour:
  Variable reward triggers on every 5th success (fixed ratio).
  This is not a variable schedule.

Fix the variable schedule logic exactly:

  After owner taps "Release cue command":
    1. Show a short countdown (random 1-2 seconds)
    2. When countdown ends, run this logic:
         const reward = Math.random() < 0.49
    3. If reward === true (49% chance):
         Show: "Good reward — good boy! 🎉"
         Show reward animation (treat/play/praise icon pulses)
    4. If reward === false (51% chance):
         Show: "Hold reward — not this time"
         Show a gentle "X" or hold indicator
    5. Either way: wait the inter-trial timer (random 7-12s)
       then show next rep

  Each trial is independently random. No memory of previous
  outcomes. Not a pattern, not a ratio tracker — pure random
  with 49% probability each time.

  The 49/51 split is deliberate. Slightly more withholding
  than rewarding builds the strongest variable schedule
  reinforcement.

  Style: ghost button (outline, no fill), smaller than HOLD
  Label: "Reset"

    current_points = command's current remaining points
    deduction = Math.floor(current_points * 0.25)
