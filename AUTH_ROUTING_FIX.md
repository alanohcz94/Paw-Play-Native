# Auth Routing Fix — `invalid_request` on Sign In

## Problem

Clicking **Sign In** threw an `invalid_request` error from Replit's OIDC provider.

## Root Cause

In `artifacts/pawplay/lib/auth.tsx`, `AuthSession.makeRedirectUri()` was called without an explicit scheme:

```typescript
// Before — generates different URIs per environment
const redirectUri = AuthSession.makeRedirectUri();
```

| Environment     | URI generated           | Registered with Replit? |
|-----------------|-------------------------|-------------------------|
| Expo Go         | `exp://u.expo.dev/...`  | No → **invalid_request** |
| Dev/Prod build  | `pawplay://`            | Needs to be registered  |
| Web             | Based on current origin | No                      |

Replit's OIDC server rejects any `redirect_uri` that isn't pre-registered for the client, returning `invalid_request`.

## Fix Applied

`artifacts/pawplay/lib/auth.tsx` line 52 — explicit scheme and path:

```typescript
// After — consistent redirect URI across all environments
const redirectUri = AuthSession.makeRedirectUri({
  scheme: "pawplay",
  path: "auth",
});
// Generates: pawplay://auth
```

This produces `pawplay://auth` consistently regardless of whether you're running in Expo Go, a dev build, or a production build.

## EAS Config (`eas.json`)

`EXPO_PUBLIC_REPL_ID` was correctly added to all three profiles (development, preview, production). This value maps to the OAuth `client_id` used in `getClientId()` inside `auth.tsx`. No changes needed here.

```json
"env": {
  "EXPO_PUBLIC_REPL_ID": "5e723a64-9f24-4eb1-8254-87b49171d7cb",
  "EXPO_PUBLIC_DOMAIN": "paw-play-native.replit.app"
}
```

## Required: Register Redirect URI in Replit

The code fix alone is not enough. You must register `pawplay://auth` as an authorized redirect URI in your Replit project's OAuth settings:

1. Go to your Replit project → **Settings** → **OAuth/OIDC**
2. Add `pawplay://auth` to the list of **Authorized redirect URIs**
3. Save

Without this step, Replit's OIDC server will still reject the request.

## Local Development Without Replit

The `dev` script in `package.json` relies on Replit environment variables (`$REPL_ID`, `$REPLIT_DEV_DOMAIN`). When developing outside Replit, create a `.env.local` file in `artifacts/pawplay/`:

```env
EXPO_PUBLIC_REPL_ID=5e723a64-9f24-4eb1-8254-87b49171d7cb
EXPO_PUBLIC_DOMAIN=paw-play-native.replit.app
```

> `.env.local` is already gitignored by Expo's default `.gitignore`.

## Auth Flow Summary

```
User taps "Sign In"
  → login() → promptAsync()
  → Opens browser: Replit OIDC authorization URL
      client_id = EXPO_PUBLIC_REPL_ID
      redirect_uri = pawplay://auth   ← fixed
      PKCE code_challenge (auto-generated by expo-auth-session)
  → User authenticates in browser
  → Browser redirects to pawplay://auth?code=...
  → App receives deep link → expo-auth-session picks it up
  → POST /api/mobile-auth/token-exchange (code + code_verifier)
  → Token stored in expo-secure-store
  → GET /api/auth/user → user state set → navigate to (tabs)
```
